Module: $mod$ | Topic: $topic$ | Calculated Fields

In the Eval topic earlier on in the workshop, you learned how to create fields inline using calculations. Eval expressions can be complex, and if you need to use a long and complex eval expression on a regular basis, retyping the expression accurately can be tedious.

Instead, you can use calculated fields as a shortcut for performing repetitive, long, or complex transformations using the eval command.

Calculated fields enable you to define fields with eval expressions. When writing a search, you can cut out the eval expression and reference the field like any other extracted field. The fields are extracted at search time and added to events that include the fields in the eval expressions.

We are going to create a calculated field for the same mbytes definition you performed manually in Example 2 of the Eval topic.

  • 1. Create Calculated Field
  • 2. Test Calculated Field

  • Click “Add new” to the right of the “Calculated fields”


  • Make sure the “Destination app” is SplunkSearchPartyWorkshop
  • Make sure “Apply to” is set to sourcetype, and in the “named” box you type access_combined. (Make sure you spell access_combined correctly !!)
  • In the "Name" box, type mbytes
  • In the "Eval expression" box, type bytes/1024/1024
  • Click the “Save” button

  • Go to search by first selecting "Apps" -> "Search Party Workshop"


  • Then click on "Search", which is located on the upper left side just next to "Start Here".

  • Using a timeframe of October 1, 2016, type the following in the search bar:

index=$idx$ sourcetype=$stype$

  • If the calculated field worked, you will see a new field called mbytes under the interesting fields on the left hand side.
